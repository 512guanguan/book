# 小说章节撰写指导

## 一、工作流程（必做步骤）

### Step 1：收集上下文信息

撰写前必须读取以下文件获取最新设定：

```
大纲/行文规范.md          # 核心行文规范、幽默风格、对话模板
大纲/主角设定.md          # 主角人设、穿越背景、思维特点
大纲/修炼体系.md          # 境界划分、世界观设定
大纲/灵海大模型.md        # 金手指设定、阶段划分（当前为建网期）
大纲/佣兵团.md            # 野草团成员设定、关系网络
大纲/势力架构.md          # 势力分布、敌人设定
大纲/渔阳城设定.md        # 当前剧情所在城市
大纲/常见错误记录.md      # 已犯错误清单，避免重蹈覆辙
大纲/资源经济.md          # 货币体系、物价标准
```

同时读取**最近2-3章正文**，确保剧情衔接、人物状态连续。

### Step 2：确定章节定位

明确本章在故事中的位置：
- 当前处于哪个阶段？上一章结尾的钩子是什么？
- 本章需要推进什么？（修为/情报/关系/爽点）
- 本章的小爽点是什么？
- 本章结尾钩子是什么？

### Step 3：思考并设计剧情，撰写正文

章节字数3000字左右。每章至少推进一项，每章结尾留钩子。

### Step 4：强制复核（不可跳过）

> ⚠️ **此步骤为强制步骤，严禁跳过。** 写完正文后必须逐项复核，复核不通过的章节必须修改至通过后才能交付。

#### 4.1 事实与设定复核

逐条核对，任何一条不通过都必须修改：

- [ ] **金手指状态**：灵海大模型当前处于建网期，未联网，无任何推理/推演/计算能力。是否误用了"大模型分析""灵海推演""灵海提示"等表述？
- [ ] **混沌保密**：混沌塔、混沌真解、混沌灵根、混沌气息等概念，是否对外人泄露？主角对外只说"自己琢磨的""运气好"？
- [ ] **NPC认知**：低阶NPC是否识别出了混沌属性？丹药评价是否只用了"成色好""杂质少"等通用标准？
- [ ] **货币单位**：灵石/灵币是否混用？1下品灵石=1000两灵币的换算是否正确？物价是否符合 `资源经济.md`？
- [ ] **灵海与识海**：危险感知是否归因于识海/神识？是否错误归因于灵海主动"示警""分析"？
- [ ] **储物设定**：主角是否用混沌塔储物？是否出现了主角用物理容器装东西的矛盾？
- [ ] **修为境界**：当前练气2层，能力表现是否符合练气2层水平？是否有越级表现却无合理解释？
- [ ] **妖兽等级**：妖兽是否用"凡兽/灵兽/地兽/天兽+初/中/后"描述？是否误用了"练气X层的妖兽"等修士境界指代妖兽？
- [ ] **进塔机制**：进塔时是否写"整个人凭空消失"？是否误写"身体留在原地""叫不醒推不动"？
- [ ] **炼丹障眼法**：陆渊对外炼丹是否维持了"屋里丹炉+屏蔽阵盘"的障眼法？是否有团里人对炼丹场景产生合理疑问却未解答？
- [ ] **人物状态**：与上一章结尾的人物状态是否衔接？修为、伤势、持有物品、人际关系是否连续？
- [ ] **剧情衔接**：与上一章结尾的钩子是否衔接？是否有未交代的跳跃？

#### 4.2 逻辑与剧情复核

- [ ] **逻辑通顺**：事件因果是否成立？人物行为动机是否合理？是否有"为推进剧情而强行"的桥段？
- [ ] **剧情合理**：冲突设置是否合理？解决方式是否水到渠成？是否有明显的降智或巧合？
- [ ] **反派逻辑**：反派行为是否符合其利益和性格？是否为凸显主角而强行降智？
- [ ] **信息密度**：本章是否推进了至少一项（修为/情报/关系/爽点）？是否有注水或重复？

#### 4.3 文风与AI味复核

- [ ] **AI味检测**（重点）：
  - 是否有"信息整理：1.2.3.4."式条目罗列代替叙事？
  - 是否有过于工整、机械的段落结构（如"首先…其次…最后…"）？
  - 是否有不符合人物身份的书面化、说明性语言？
  - 是否有"值得一提的是""不仅如此""总而言之"等AI常用过渡词？
  - 是否有大段设定科普而非融入剧情？
  - 情绪表达是否扁平、缺少层次？
  - → 以上任何一条命中，都必须重写相关段落
- [ ] **幽默自然度**：幽默是否符合情境？是否尬？程序员梗是否控制在每章1-2处？
- [ ] **英语/术语检查**：正文中是否出现了Bug、Debug、AI、GPU、CPU、CNN、Transformer、root、VIP、Beta、TODO等英语词汇？是否用了太专业的程序员术语？→ 必须改为通俗中文表达
- [ ] **主角说话风格（强制）**：主角是否"端着"了？面对撩拨/挑衅/煽情是否只会脸红后退、"你别闹"、"……"？是否有"鼻子一酸""感动流泪"这种过度煽情？→ 必须改为"接住+反撩/反将一军"或"用玩笑化解"。主角是贱嘴乐天派，不是端着的正经人
- [ ] **系统流感**：是否写得像带系统的修仙文？（金手指应通过小紫互动和主角自身思考体现，而非系统弹窗式提示）
- [ ] **对话质量**：每句对话是否有信息量或性格？不同人物说话方式是否区分？是否有"他说""她道"等冗余标签？
- [ ] **描写质量**：战斗/修炼描写是否嵌在剧情中？是否有独立堆砌的设定段落？

#### 4.4 结构与节奏复核

- [ ] **字数**：是否在3000字左右（关键章节可3500字）？
- [ ] **爽点**：是否有至少一个小爽点或信息揭露？
- [ ] **钩子**：结尾是否留了钩子？钩子是否能在下一章或近两章兑现？
- [ ] **主线关联**：是否与主线（回家/佣兵团发展/后期家仇/天渊之谜）有关联？

#### 4.5 常见错误终检

对照 `常见错误记录.md` 逐条核对，确保不犯已知错误。

#### 4.6 复核结果处理

- **全部通过**：可交付章节
- **存在不通过项**：必须修改后重新复核，直到全部通过
- **无法确定的问题**：向用户说明并征求意见，不擅自决定

## 四、金手指当前状态（重要）

### 灵海大模型

> **当前阶段：建网期（练气2层）**

**能做的**：
- 节点自动吸收灵气，修炼速度比普通人快
- 受伤恢复更快
- 灵力调用更精准

**绝对不能做的**（建网期一律不具备）：
- ❌ 推演战斗
- ❌ 分析功法破绽
- ❌ 预判对手动作
- ❌ 任何需要"计算""推理"的能力
- ❌ 灵海主动"示警""分析"

> 灵海大模型没有版本号，是逐步进化的系统。神识接入网络（联网）后才具备推理能力，当前尚未联网。

### 神识异常（当前特殊设定）

正文第4-5章已建立：陆渊的神识增长异常快（感知30丈），原因尚未明确（小紫推测与灵根有关）。这是**尚未解释的伏笔**，不是灵海大模型的能力。后续可通过灵根特殊性或混沌塔影响来圆。

### 混沌塔

- 第一层（混沌大厅）：无限储物，可随时进出
- 第二层（混沌世界）：3倍时间加速修炼
- 第三层（万药园）：种药、炼丹、查阅典籍
- **进塔机制**：进塔时**整个人凭空消失**（原地没了），出塔时原地出现。不是"身体留在原地"
- **限制**：进塔时人凭空消失有风险（若被人目击消失会暴露异常）；空间被封锁时无法进入；外界无法攻击塔身（塔在灵魂里）
- **保密**：混沌塔是天渊至宝，绝不能对外人透露半个字

### 炼丹障眼法（重要逻辑约束）

陆渊真正的炼丹在混沌塔万药园内进行，但对外必须维持"在屋里炼丹"的假象：

- **二手丹炉**：阿福从旧货铺买的磕角青铜炉，摆在屋里做样子。偶尔烧废弃灵草边角料制造药味
- **屏蔽阵盘**：铺在床下，遮住灵气波动（虽然塔里炼丹外面没波动，但阵盘让"闻不到药味"更合理）
- **进塔时机**：趁大壮他们在院里干活、动静大的时候进塔，出来后再在破丹炉前坐会儿
- **对外说辞**：炼丹药味大，灵气波动明显，怕引来妖兽和城防营，所以用阵盘遮住

> 写作时务必保持这个障眼法逻辑一致，不要写出"陆渊在屋里明目张胆炼丹被闻到"或"团里人看到丹炉却没有疑惑"等矛盾。

## 五、章节结构规范

### 字数控制

- 单章3000字左右，不拖沓
- 关键章节可延长至3500字

### 信息密度

每章必须推进至少一项：
- 修为提升
- 情报获取
- 关系发展
- 爽点爆发

### 节奏控制

- 单章1个场景为主，1个小冲突或1个信息揭露
- 每章结尾留钩子，且钩子必须在下一章或近两章内兑现
- 修炼/炼丹描写嵌在剧情中，不写纯修炼章

## 六、主角人设执行

### 陆渊的性格定位：贱嘴乐天派

主角是"又贱又靠谱的乐天派"——嘴上不饶人，让人又爱又恨想揍他；关键时刻能扛事，仗义起来比谁都稳。参考《修真聊天群》宋书航的贫嘴、《逆天邪神》云澈少年期的痞气、《斗破苍穹》萧炎的嘴硬。

### 说话方式（核心约束）

1. **嘴贱**：吐槽是本能，接梗是反应，被揍了也要嘴硬，输了也要嘴上占便宜。让人又爱又恨想揍他
2. **乐天派**：遇事先调侃两句再想办法，越是绝境越要嘴硬（嘴硬是他保持冷静的方式）
3. **自嘲不手软**：穷、废灵根、送外卖、单身狗……什么都能拿来调侃自己
4. **面对撩拨/挑衅/煽情必须接得住**：不许只会脸红后退、"你别闹"、"……"。要接住+反将一军。这是强制原则
5. **对女角色不退缩**：面对暧昧不害羞退让，而是接住+反撩。成年人间的互动有分寸但不退缩
6. **核心反差**：嘴上不正经，做事很靠谱。平时吊儿郎当满嘴跑火车，关键时刻永远挡在前面——事后还要嘴硬"我可不是为了你"

### 思维特点

| 身份 | 思维特点 | 应用场景 |
|------|---------|---------|
| 程序员 | 拆解问题、找规律、优化流程 | 功法学习、炼丹调参、商业布局 |
| 外卖员 | 时间敏感、空间感知、抗压处理 | 秘境探索、任务执行、危机应对 |

> 注意：当前处于建网期，程序员思维体现在**学习速度快、善于观察总结、追求效率**，而非"大模型推演"。不能用灵海大模型做推理，只能靠脑子自己想。

### 程序员梗控制

- 每章最多1-2处，别变成段子集
- 自然融入情境，不强行抖机灵
- 常见形式：内心吐槽（"这不就是XXX吗"）、比喻（"调火候""找毛病""复利"）

### 说话风格对照表（写作时参照）

| 场景 | ❌ 禁止写法 | ✅ 推荐写法 |
|------|-----------|-----------|
| 被小紫撩 | 脸红、"你别闹"、"……" | 接住+反撩，"好看是好看，就是没丹炉好看" |
| 被看穿实力 | 慌张、沉默 | 嘴硬，"你看错了，我这是练气二层的正常水平，天才型选手" |
| 被朋友关心 | 鼻子一酸、感动流泪 | 用玩笑化解，"想把我喂成猪好让我跑不动？" |
| 遇到危机 | 严肃分析、"我必须冷静" | 先吐槽一句再想办法 |
| 被反派威胁 | 正义凛然、"你休想" | 嘴上占便宜，"你这么凶，家里人是没给你吃饱饭？" |
| 谈合作 | 像大老板谈判 | 带着贱劲，"卖丹药可以，卖身不行" |

## 七、塔灵（小紫）人设执行

### 性格

- **身材火爆的御姐**：漂亮、妩媚、慵懒、丰乳肥臀
- **爱调侃**：喜欢逗陆渊，用"小家伙""笨蛋""亲爱的""宝宝"等各种称呼
- **关键时刻正经**：危险时给出靠谱建议
- **记忆残缺**：忘了原名，自称小紫，偶尔会陷入恍惚
- **有女主潜力**：与陆渊关系会逐步深化

### 与陆渊的互动模式（核心约束）

**互怼互撩**，不是"小紫撩+陆渊退"。陆渊必须接得住、还能反将一军：

- 小紫撩→陆渊接住+反撩→小紫笑骂→关键时刻正经
- 陆渊的反撩要自然、有分寸、带着调侃的劲儿，不猥琐不上头
- 成年人之间的暧昧互动可以有尺度：言语上的互相占便宜、肢体上的自然接触（不经意碰到手、凑近说话）、眼神拉扯

## 九、场景描写规范

### 描写原则

1. **少而精**：关键场景详写，过渡一笔带过
2. **感官为主**：视觉/听觉/触觉，少抽象形容
3. **服务剧情**：描写推动氛围或暗示信息

### 战斗描写模板

```
[起] 双方对峙，一句话点明实力差距
[承] 交手数招，主角处于下风或试探
[转] 主角使出底牌/利用优势破局
[合] 一击决胜，旁观者震惊视角收尾
```

> 当前主角实力弱，战斗以"智取""逃跑""借势"为主，少写正面硬刚。

## 十、常见错误防范（对照常见错误记录.md）

### 1. 混沌塔/混沌相关概念对外泄露
- 混沌塔、混沌真解、混沌气息、混沌灵根——**绝不对外人提**
- 对外人只说"自己琢磨的""天赋""运气好"

### 2. 普通NPC识别混沌属性
- 低阶修士不可能认识混沌气息
- NPC评价丹药只用"成色好""杂质少""药香纯"等通用标准

### 3. 货币单位混乱
- 灵石（修仙界用）与灵币（凡人界用）不同，1下品灵石=1000两灵币
- 写价格前查 `资源经济.md`

### 4. 灵海与识海功能混淆
- 灵海（丹田）：存灵力，能量相关
- 识海（眉心）：存神识，感知/分析相关
- 危险感知归因于识海/神识，不归因于灵海

### 5. 储物设定矛盾
- 主角用混沌塔储物，不用物理容器
- 储物袋是通用工具但有价格门槛（最便宜10块下品灵石）

### 6. 系统流式信息罗列
- 不用"信息整理：1.2.3.4."式条目代替叙事
- 信息通过角色思考和对话自然呈现
- 系统流写法破坏沉浸感，显得AI味重

### 7. 灵海大模型能力越界
- 当前是建网期，**没有任何推理/推演/计算能力**
- 不能写"灵海大模型分析""大模型推演""灵海给出提示"
- 灵海里的光点/网络只是伏笔，主角目前不理解它

### 8. 妖兽等级用修士境界描述
- 妖兽等级用"凡兽/灵兽/地兽/天兽+初/中/后"，**不用"练气X层"指代妖兽**
- 正确示例：✅"凡兽中期的妖兽" ❌"练气五层的妖兽"

### 9. 进塔机制矛盾
- 进塔时**整个人凭空消失**（原地没了），出塔时原地出现
- **不能写**"身体留在原地""叫不醒推不动""外界看来在屋里待了一整天"
- 进塔的风险是"人凭空消失被目击"，不是"身体留在原地被发现"

### 10. 炼丹障眼法逻辑
- 陆渊真正炼丹在混沌塔内，对外用二手丹炉+屏蔽阵盘做障眼法
- 进塔时机：趁大壮等人在院里干活时进塔，出来后在破丹炉前坐会儿
- 不能写出"团里人看到炼丹场景却无疑问"或"屋里没有丹炉却在炼丹"的矛盾

## 十一、禁忌清单

### 绝对禁止

- ❌ 纯虐主章节（主角可输，但当章或下章找回场子）
- ❌ 圣母（对敌人不留情）
- ❌ 注水（每章必须有推进）
- ❌ 重复（升级流程、打脸套路、装逼模式）
- ❌ 崩人设（性格前后一致）
- ❌ 降智反派（反派有逻辑）
- ❌ 后宫（感情线单一，目前只有友情，后续对小紫动情）
- ❌ 大段设定科普（融入剧情）
- ❌ 灵海大模型越界使用（建网期无推理能力）
- ❌ 混沌相关概念对外泄露
- ❌ 英语/专业术语泛滥：正文中不得出现Bug、Debug、AI、GPU、CPU、CNN、Transformer、root、VIP、Beta、TODO等英语词汇，用中文替代（如"找毛病""找漏洞""找错"代替Bug/Debug；"琢磨""苦练"代替调参/训练；"根基""外挂法宝"代替硬件/GPU）
- ❌ 程序员梗过度：每章最多1-2处，且必须用通俗中文表达，不堆砌专业术语
- ❌ 主角面对撩拨/挑衅/煽情只会脸红后退、"你别闹"、"……"（必须接住+反撩/反将一军）
- ❌ 主角"端着"——像大老板谈判、像中年人持重、像纯情少年害羞（他是贱嘴乐天派）
- ❌ 过度煽情——"鼻子一酸""感动流泪"（用玩笑化解煽情）

### 容易翻车的点

1. 程序员梗别过头（每章最多1-2处）
2. 小紫别太抢戏，但可以换装、大胆前卫性感
3. 灵海大模型别太强（当前建网期，无推理能力）
4. 回家执念别太丧（想念后立刻转化为行动）
5. 幽默别尬（符合情境）
6. 装逼别太早（前期藏拙为主，实力不够别硬装）
7. 金手指别万能（要有代价、有翻车）
8. 贱要有分寸——对长辈、恩人、弱者有底线，不是谁都怼
9. 反撩要有度——成年人间的暧昧互动，不猥琐不上头，有分寸感
