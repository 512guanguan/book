# 小说章节撰写指导（整合约束版）

> 本文件由「工作目录 skill」与「仓库 .trae skill」整合而成，并以 `大纲/行文规范.md` 为权威约定来源。所有"硬约束"（字数、破折号、违禁词、金手指不退化）以本文为准。

## 一、工作流程（必做步骤）

### Step 1：收集上下文信息

撰写前必须读取以下文件获取最新设定：

```
大纲/行文规范.md          # 核心行文规范、幽默风格、对话模板（权威约定）
大纲/主角设定.md          # 主角人设、穿越背景、思维特点
大纲/修炼体系.md          # 境界划分、世界观设定
大纲/灵海大模型.md        # 金手指设定、阶段划分（当前联网期·筑基中期）
大纲/佣兵团.md            # 野草团成员设定、关系网络
大纲/势力架构.md          # 势力分布、敌人设定
大纲/渔阳城设定.md        # 当前剧情所在城市
大纲/常见错误记录.md      # 已犯错误清单，避免重蹈覆辙
大纲/资源经济.md          # 货币体系、物价标准
大纲/渔阳城剧情大纲.md    # 14-35章详细规划
```

同时读取**最近2-3章正文**，确保剧情衔接、人物状态连续。

### Step 2：确定章节定位

明确本章在故事中的位置：
- 当前处于哪个阶段？上一章结尾的钩子是什么？
- 本章需要推进什么？（修为/情报/关系/爽点）
- 本章的小爽点是什么？
- 本章结尾钩子是什么？

### Step 3：思考并设计剧情，撰写正文

章节字数 **2500–3500 字（去空白计数）**，关键章节可到 3500 字上限，不拖沓也不注水。每章至少推进一项，每章结尾留钩子。

### Step 4：强制复核（不可跳过）

> ⚠️ **此步骤为强制步骤，严禁跳过。** 写完正文后必须逐项复核，复核不通过的章节必须修改至通过后才能交付。

#### 4.1 事实与设定复核

逐条核对，任何一条不通过都必须修改：

- [ ] **金手指状态**：灵海大模型当前已联网（**筑基中期·180节点·神识三百丈**），具备推演丹方、拆解功法、预演战斗、记忆检索能力。是否误用了超出当前能力范围的表述（如灵魂建模、法则推演等元婴/化神后才能有的能力）？是否与已建立的推演能力一致？
- [ ] **混沌保密**：混沌塔、混沌真解、混沌异火、混沌之力等概念，是否对外人泄露？主角对外只说"自己琢磨的""运气好"
- [ ] **混沌灵根未确定**：混沌灵根尚未被主角/小紫明确确认，只存在异火传递的"与你同源"信息中。正文不能写"主角的混沌灵根"作为已知事实，只能写"疑似""可能"
- [ ] **NPC认知**：低阶NPC是否识别出了混沌属性？丹药评价是否只用了"成色好""杂质少"等通用标准？
- [ ] **货币单位**：灵石/灵币是否混用？1下品灵石=1000两灵币的换算是否正确？物价是否符合 `资源经济.md`？
- [ ] **灵海与识海**：危险感知是否归因于识海/神识？是否错误归因于灵海主动"示警""分析"？
- [ ] **储物设定**：主角是否用混沌塔储物？是否出现了主角用物理容器装东西的矛盾？（注：天星石洞天、储物戒等容器是剧情道具，与"用混沌塔本体储物"不冲突，但不可写"把东西塞进混沌塔空间"这类暴露表述）
- [ ] **修为境界**：当前**筑基中期（第23章突破，第26章再破境）**，能力表现是否符合？灵海节点180（第19章被混沌异火优化、后续增长）
- [ ] **混沌异火保密**：混沌异火是绝对机密，对外炼丹只能说"用橙丹火"。异火能变色、能模拟任何属性丹火、能作为攻击手段——这些都不能对外展示
- [ ] **妖兽等级**：妖兽是否用"凡兽/灵兽/地兽/天兽+初/中/后"描述？是否误用了"练气X层的妖兽"等修士境界指代妖兽？
- [ ] **进塔机制**：进塔时是否写"整个人凭空消失"？是否误写"身体留在原地""叫不醒推不动"？天星石洞天（药谷陨石天然洞天）入口为**银色水镜裂隙**，可带人进、隔音隐蔽，与"混沌塔本体"是两套独立空间，不要混为一谈
- [ ] **炼丹障眼法**：陆渊对外炼丹是否维持了"屋里丹炉+屏蔽阵盘"的障眼法？（注：第15章后主角已在聚宝阁建立合作，障眼法主要针对团里人）
- [ ] **人物状态**：与上一章结尾的人物状态是否衔接？修为、伤势、持有物品、人际关系是否连续？
- [ ] **剧情衔接**：与上一章结尾的钩子是否衔接？是否有未交代的跳跃？
- [ ] **小紫的认知边界**：小紫长期沉睡，对天渊大陆当前情况不熟悉，不能让她说出外界具体势力、人物、水平等信息。她只知道塔内的事

#### 4.2 逻辑与剧情复核

- [ ] **逻辑通顺**：事件因果是否成立？人物行为动机是否合理？是否有"为推进剧情而强行"的桥段？
- [ ] **剧情合理**：冲突设置是否合理？解决方式是否水到渠成？是否有明显的降智或巧合？
- [ ] **反派逻辑**：反派行为是否符合其利益和性格？是否为凸显主角而强行降智？
- [ ] **信息密度**：本章是否推进了至少一项（修为/情报/关系/爽点）？是否有注水或重复？
- [ ] **灵海大模型描写模糊化**：灵海大模型的内部机制不要写得太具体（容易经不起推敲），多聚焦功能结果，少写机制原理。用"感觉""隐约""门道"等模糊表述，不写精确机制

#### 4.3 文风与AI味复核（重点）

- [ ] **自然化优先（核心要求）**：正文必须像人写的网络小说，不要像机器说明书。禁止用"首先…其次…最后…"的列举式推进；禁止"简单来说/值得注意的是/值得一提的是/不仅如此/总而言之/一方面…另一方面/具体而言/可以分为/其核心/换言之/也就是说/本质上/一般来说/通常来说/所谓"等说明文腔过渡词。用画面、动作、对话承载信息，不用抽象总结代替场景
- [ ] **禁止"列计划再逐项实现"（行文大忌·核心要求）**：是否出现"先罗列待办清单（A、B、C、D）、后逐项执行"的结构？（典型病句："优先级排得清清楚楚：X、Y、Z""他给自己排了条线：A、B、C、D，末了一举破境"）这是代码思维不是文学。所有计划性内容必须拆散，随剧情自然带出——想到就做，做完自然引出下一件，不可预告清单。修炼段可写上帝视角的路线与收获，但绝不可"先列四项再逐项打卡"
- [ ] **AI味检测**：
  - 是否有"信息整理：1.2.3.4."式条目罗列代替叙事？
  - 是否有过于工整、机械的段落结构（如"首先…其次…最后…"）？
  - 是否有不符合人物身份的书面化、说明性语言？
  - 是否有"值得一提的是""不仅如此""总而言之""此外""然而"等AI常用过渡词？→ 删掉或改成自然衔接
  - 是否有大段设定科普而非融入剧情？
  - 情绪表达是否扁平、缺少层次？
  - 是否有破折号（——）过度使用？**单章超过1处即违规**（硬约束：每章≤1个，能不用就不用）→ 仅保留"对话被打断"和"强戏剧性停顿"两种场景，其余改句号/逗号/冒号
  - 是否有否定式排比（"不仅……而且……""不是……而是……""这不仅仅是……"）？→ 改为直接陈述
  - 是否有三段式法则（强行凑三个并列项"X、Y和Z"）？→ 改为两项或四项，或拆成句子
  - 是否有刻意换词（同义词循环，如"主人公→主要角色→中心人物"指同一人）？→ 统一用词，重复不可怕
  - 是否有填充短语（"为了实现这一目标""在这个时间点""值得注意的是"）？→ 删掉，直接说
  - 是否有过度限定（叠甲式"可能潜在地或许""似乎大概也许"）？→ 一个"可能"够用
  - 是否有宣传性语言（"充满活力的""丰富的""令人叹为观止的"等夸张形容）？→ 换成具体细节
  - 是否有以"着"结尾的肤浅分析（"彰显着""体现着""反映着""象征着"等虚假深度）？→ 删掉或改成具体动作
  - 是否有系动词回避（用"作为/代表/标志着"代替简单的"是"）？→ 直接用"是"
  - 是否有模糊归因滥用（"据说""传闻""有人认为""观察者指出"）？→ 给出具体来源或删掉
  - 是否有通用积极结论（结尾过于圆满、"未来可期"式收尾）？→ 留悬念或写具体计划
  - → 以上任何一条命中，都必须重写相关段落
- [ ] **幽默自然度**：幽默是否符合情境？是否尬？程序员梗是否控制在每章1-2处？
- [ ] **英语/术语检查（硬约束）**：正文中是否出现了 **Bug、Debug、AI、GPU、CPU、CNN、Transformer、root、VIP、Beta、TODO** 等英语词汇？是否用了太专业的程序员/科学术语（**算法、调参、代码、参数、算力、变量、数据、版本号、BOSS、等级压制、编译、乱码、加密**）？→ 必须改为通俗中文表达（如"找毛病"代Bug、"琢磨/苦练"代调参、"根基/外挂法宝"代硬件）。注意：**「灵海大模型」是金手指专名，允许使用，不算违禁**
- [ ] **外卖相关词（硬约束）**：是否出现 bare「外卖」（指餐/订单的名词）、「送外卖」、「跑外卖」？→ 一律禁用。主角身份用**「外卖员」**（允许），动作可用**「送餐」「跑腿」**。例："死在送外卖的路上"→"死在送餐的路上"；"以前真是送外卖的？"→"以前真是外卖员？"
- [ ] **混沌专名（硬约束）**：「混沌」只能作为专名出现（混沌塔/混沌真解/混沌异火/混沌之力/混沌灵根），不可当作普通形容词随意使用
- [ ] **主角说话风格（强制）**：主角是否"端着"了？面对撩拨/挑衅/煽情是否只会脸红后退、"你别闹"、"……"？是否有"鼻子一酸""感动流泪"这种过度煽情？→ 必须改为"接住+反撩/反将一军"或"用玩笑化解"。主角是贱嘴乐天派，不是端着的正经人
- [ ] **系统流感**：是否写得像带系统的修仙文？（金手指应通过小紫互动和主角自身思考体现，而非系统弹窗式提示）
- [ ] **对话质量**：每句对话是否有信息量或性格？不同人物说话方式是否区分？是否有"他说""她道"等冗余标签？
- [ ] **描写质量**：战斗/修炼描写是否嵌在剧情中？是否有独立堆砌的设定段落？
- [ ] **废话控制**：引出新设定/新剧情时不要啰嗦闲聊，该直接就直接。小紫引出异火这种关键剧情，三句话引完，不要铺垫闲聊
- [ ] **格式检查**：正文中是否出现了章节标题（"# 第X章 标题"）、`---` 场景分隔线、或"（第X章完）""本章完"等标记？→ 全部删除，正文只保留纯叙事内容，标题和分隔由文件系统管理
- [ ] **行文风格（参照《逆天邪神》）**：句子是否长短交错（不全是短句堆砌）？描写是否有画面感和冲击力（用具体细节代替抽象形容）？修炼突破是否有视觉触感声音？装逼是否通过旁人反应和环境变化渲染？节奏是否明快（有资源赶紧用，有瓶颈赶紧冲，不为凑字数拖延）？

#### 4.4 结构与节奏复核

- [ ] **字数**：是否在 **2500–3500 字（去空白）** 区间？低于2500偏水、高于3500拖沓（关键章上限3500）
- [ ] **爽点**：是否有至少一个小爽点或信息揭露？
- [ ] **钩子**：结尾是否留了钩子？钩子是否能在下一章或近两章内兑现？
- [ ] **主线关联**：是否与主线（回家/佣兵团发展/后期家仇/天渊之谜）有关联？
- [ ] **破折号（——）**：每章是否 **≤1 个**？能不用就不用？句中停顿是否用逗号、句号或自然断句替代了破折号？

#### 4.5 常见错误终检

对照 `常见错误记录.md` 逐条核对，确保不犯已知错误。

#### 4.6 复核结果处理

- **全部通过**：可交付章节
- **存在不通过项**：必须修改后重新复核，直到全部通过
- **无法确定的问题**：向用户说明并征求意见，不擅自决定

## 六、主角人设执行

### 陆渊的性格定位：贱嘴乐天派

主角是"又贱又靠谱的乐天派"——嘴上不饶人，让人又爱又恨想揍他；关键时刻能扛事，仗义起来比谁都稳。参考《修真聊天群》宋书航的贫嘴、《逆天邪神》云澈少年期的痞气、《斗破苍穹》萧炎的嘴硬。

### 说话方式（核心约束）

1. **嘴贱**：吐槽是本能，接梗是反应，被揍了也要嘴硬，输了也要嘴上占便宜
2. **乐天派**：遇事先调侃两句再想办法，越是绝境越要嘴硬
3. **自嘲不手软**：穷、废灵根、**外卖员**、单身狗……什么都能拿来调侃自己（注意：用"外卖员"身份词，不用"送外卖/跑外卖"动词）
4. **面对撩拨/挑衅/煽情必须接得住**：不许只会脸红后退、"你别闹"、"……"。要接住+反将一军
5. **对女角色不退缩**：面对暧昧不害羞退让，而是接住+反撩
6. **核心反差**：嘴上不正经，做事很靠谱

### 思维特点

| 身份 | 思维特点 | 应用场景 |
|------|---------|---------|
| 程序员 | 拆解问题、找规律、优化流程 | 功法学习、炼丹、商业布局 |

### 程序员梗控制

- 每章最多1-2处，别变成段子集
- 自然融入情境，不强行抖机灵
- 用通俗中文表达，不用英语/专业术语（Bug/算法/代码/参数/数据/算力 等一律禁用）

## 九、场景描写规范

### 描写原则

1. **少而精**：关键场景详写，过渡一笔带过
2. **感官为主**：视觉/听觉/触觉，少抽象形容
3. **服务剧情**：描写推动氛围或暗示信息

### 战斗描写模板

```
[起] 双方对峙，一句话点明实力差距
[承] 交手数招，主角处于下风或试探
[转] 主角使出底牌/利用优势破局
[合] 一击决胜，旁观者震惊视角收尾
```

### 行文风格规范（参照《逆天邪神》）

> 本小说行文风格以《逆天邪神》为主要参照，写正文时必须遵循以下要点：

1. **叙事节奏明快**：该快的地方一笔带过（赶路、日常、过渡），该慢的地方细写（关键战斗、突破、情绪高潮、装逼时刻）。不拖泥带水，不注水凑字数
2. **句子长短交错**：不全是短句堆砌（那是水文），也不全是长句（那是论文）。关键动作用短句冲击，描写渲染用中长句铺开。一段话里要有节奏变化
3. **对话简洁有锋芒**：每句话要么推进信息，要么展现性格。不说废话，不解释自己说过的话。人物之间的对话要有来有回，有交锋感
4. **描写重画面感和冲击力**：用具体感官细节代替抽象形容。不写"很强"，写"一剑下去山头平了"。不写"很痛"，写"像被烧红的铁棍捅进骨头里"
5. **修炼突破有画面感**：写灵力运转像写水流，写突破像写爆炸。有视觉、有触感、有声音。不干巴巴地列数据说"灵力涨了三成"
6. **装逼有气势**：通过旁人反应、环境变化、动作细节来渲染，不是干巴巴地说"他很厉害"。装逼的时候画面要拉满，节奏要到位
7. **情绪到位但不煽情**：该冷的时候冷，该燃的时候燃。不强行催泪，不鼻子一酸，不感动流泪。情绪通过行为和细节传递，不通过形容词堆砌
8. **主角内心独白有痞气**：想到什么一两个短句带过，不长篇大论地分析。有锋芒，有态度，但不啰嗦
9. **场景转换流畅**：用自然过渡或空行，不拖泥带水。不写"不知不觉过了三天"这种废话，直接"三天后"
10. **该墨迹的地方不墨迹，不该墨迹的地方绝不墨迹**：修炼突破、关键决策、爽点爆发要干脆利落。有资源就赶紧用，有瓶颈就赶紧冲，有仇就赶紧报。不要为了凑字数而拖延节奏
11. **禁止"列计划再逐项实现"（行文大忌）**：不得先罗列主角接下来的待办清单（如"优先级排得清清楚楚：确认潮信、备丹药、拔人、磨底牌"或"他给自己排了条线：备丹、淬体、壮魂、磨招"），再在下文逐项执行。这是写代码、做项目的思维，不是文学创作。小说内容必须随剧情自然带出——主角想到什么便去做，做到一处自然引出下一处，读者跟着角色走，而不是跟着一张待办清单走。修炼、备战等"多步骤"段落是重灾区，务必用自然推进代替预先罗列。修炼段写上帝视角的"路线与收获"可以，但绝不可写成"先列四项再逐项打卡"

## 十、常见错误防范

### 1. 混沌相关概念对外泄露
- 混沌塔、混沌真解、混沌异火、混沌之力——**绝不对外人提**
- 「混沌」只能作专名，不可当普通形容词

### 2. 混沌灵根当已知事实
- 混沌灵根尚未确认，只存在异火的"与你同源"信息中
- 正文不能写"主角的混沌灵根"作为已知事实

### 3. 小紫知道外界情况
- 小紫长期沉睡，对天渊大陆当前势力/人物/水平不熟悉
- 不能让小紫评价"东江城丹师工会水平"等外界具体情况

### 4. 灵海大模型机制写太细
- 内部机制不要掰太细，容易经不起推敲
- 多聚焦功能结果，少写机制原理
- 用"感觉""隐约""门道"等模糊表述
- **「灵海大模型」本身是金手指专名，可正常使用**

### 5. 货币单位混乱
- 灵石（修仙界用）与灵币（凡人界用）不同

### 6. 灵海与识海功能混淆
- 灵海（丹田）：存灵力
- 识海（眉心）：存神识，感知/分析

### 7. 储物设定矛盾
- 主角用混沌塔本体储物需隐蔽；天星石洞天、储物戒为剧情道具，可正常使用，但不可写"把东西塞进混沌塔空间"这类暴露表述

### 8. 系统流式信息罗列
- 不用条目罗列代替叙事
- 不用"首先…其次…最后…"的说明书腔推进

### 9. 妖兽等级用修士境界描述
- 用"凡兽/灵兽/地兽/天兽+初/中/后"

### 10. 进塔机制矛盾
- 整个人凭空消失，不是身体留原地
- 天星石洞天入口为银色水镜裂隙（独立空间，可带人），与混沌塔本体分开

### 11. 英语/专业术语泛滥（硬约束）
- 不出现 Bug、Debug、AI、GPU、CPU、CNN、Transformer、root、VIP、Beta、TODO 等英语词
- 不出现 算法、调参、代码、参数、算力、变量、数据、版本号、BOSS、等级压制、编译、乱码、加密 等程序员/科学术语
- 程序员梗用通俗中文表达

### 12. 外卖相关词误用（硬约束）
- 禁用 bare「外卖」（指餐/订单）、「送外卖」、「跑外卖」
- 主角身份用「外卖员」（允许），动作可用「送餐」「跑腿」

### 13. 破折号过度使用（硬约束·AI写作特征）
- 破折号（——）过度使用是AI生成文本的典型痕迹，人类作者不会如此高频
- **单章不超过1个，能不用就不用**
- 仅保留两种场景：对话被打断（"等回来背你——"）、强戏剧性停顿/喜剧节奏
- 其余破折号按语义替换为句号/逗号/冒号
- 同段双破折号是重灾区，必须拆句

### 14. 正文中出现标题、分隔线、章节完标记
- 正文中不要写章节标题（"# 第X章 标题"），标题由文件名管理
- 不要用 `---` 作为场景分隔线，场景转换用自然过渡或空行
- 不要在结尾写"（第X章完）""本章完"等标记
- 正文只保留纯叙事内容

### 15. AI味写作模式（综合排查）
除破折号外，以下模式也需在写完后逐项排查：
- **否定式排比**："不仅……而且……""不是……而是……" → 改直接陈述
- **三段式法则**：强行凑三个并列项"X、Y和Z" → 改两项或四项，或拆成句子
- **刻意换词**：同义词循环（"主人公→主要角色→中心人物"指同一人）→ 统一用词，重复不可怕
- **填充短语**："为了实现这一目标""在这个时间点""值得注意的是" → 删掉，直接说
- **过度限定**：叠甲式"可能潜在地或许""似乎大概也许" → 一个"可能"够用
- **宣传性语言**："充满活力的""丰富的""令人叹为观止的" → 换成具体细节
- **"着"结尾虚假深度**："彰显着""体现着""反映着""象征着" → 删掉或改成具体动作
- **系动词回避**："作为/代表/标志着" → 直接用"是"
- **模糊归因**："据说""有人认为""观察者指出" → 给具体来源或删掉
- **通用积极结论**："未来可期"式收尾 → 留悬念或写具体计划
- **说明书腔**：用"首先/其次/最后/简单来说/值得注意的是/值得一提的是/具体而言/可以分为/其核心/换言之/也就是说/本质上/一般来说/通常来说/所谓"等词组织行文 → 一律改为画面/动作/对话承载
- **列计划再逐项实现（行文大忌）**：先罗列待办清单（"优先级排得清清楚楚：A、B、C、D""他给自己排了条线：A、B、C、D"）再下文逐项执行 → 这是代码思维不是文学。把清单拆散，让每件事随剧情自然带出，想到就做、做完自然引出下一件。修炼等"多步骤"段落尤需警惕，可写上帝视角路线与收获，但不可"先列四项再逐项打卡"
